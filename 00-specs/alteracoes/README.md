# 00-specs/alteracoes/

Each incremental spec iteration lives in its own dated folder here.

## Structure
```
alteracoes/
├── YYYY-MM-DD/              ← one iteration (can be multiple per day using -HHMM suffix)
│   ├── <spec-file>.md       ← changed or added specs for this iteration
│   ├── gap-analysis.md      ← auto-generated diff vs current atualizada/
│   └── consolidation.md     ← auto-generated merge summary when consolidated
└── ...
```

## Lifecycle

1. Run `/new-iteration` (or `node scripts/new-iteration.mjs <destination-repo>`) to create
   a new dated folder here.
2. Drop or edit the spec files in the new folder.
3. The orchestrator consolidates them into `00-specs/atualizada/` at the start of
   the next phase command, producing `gap-analysis.md` and `consolidation.md`.

## What NOT to do

- Don't edit files in `00-specs/originais/` (history, never overwritten).
- Don't edit files in `00-specs/atualizada/` directly (it's regenerated by the orchestrator).
- Always change specs via a new `alteracoes/YYYY-MM-DD/` folder.
